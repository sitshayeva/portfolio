Overview of Airbnb Data:¶

People's main criteria when visiting new places are reasonable accommodation and food. Airbnb (Air-Bed-Breakfast) is an online marketplace created to meet this need of people by renting out their homes for a short term. They offer this facility at a relatively lower price than hotels. Further people worldwide prefer the homely and economical service offered by them. They offer services across various geographical locations

Dataset Source¶

The dataset for this assessment the following link: http://data.insideairbnb.com/united-kingdom/england/london/2022-09-10/data/listings.csv.gz

This dataset contains information for London such as the neighborhood offering these services, room type, price,avaliabilty, reviews. This analysis will help airbnb in improving its services.

Strategies for dealing with missing data.¶

  1. If the column has less than 5% of missing values, drop the rows where these values are missing.
  2. For the columns where the missing values are between 5< THE COLUMN <60, then fix these values.
  3. If the column has more than 60% of its values missing, drop the column.

Requirements:¶

  • Minimum Threshold

  • Maximum Threshold

    Rules of Thumb for Imputing Missing Values:¶

  • For categorical columns: the mode

  • For text columns: "Unknown"

  • For integers columns: the median or the mean

  • For floats columns: the median or the mean

KEY CONCLUSIONS¶

  1. Dataset london_data.csv contains 75 columns and 69351 records (listings).
  2. Columns contain information like: id, listing_url, scrape_id, last_scraped , source, name, description, neighborhood_overview, picture_url, host_id, host_url, host_name, host_since, host_location, host_about, host_response_time, host_response_rate, host_acceptance_rate, host_is_superhost, host_thumbnail_url, host_picture_url, host_neighbourhood etc.
  3. First step of Data Cleaning is removing unwanted columns from dataset. For this Data Analysis we don't need all 75 columns. So we dropped 18 columns that we didn't plan to use for Data Analysis.
  • listing_url, scrape_id, last_scraped, source,
  • picture_url, host_url, host_name, host_location,
  • host_thumbnail_url, host_picture_url,
  • latitude, longitude, calendar_updated,
  • availability_30, availability_60, availability_90,
  • calendar_last_scraped, license.
  1. Second of Data Cleaning is handling with missing values and duplicated values. Missing values can be imputed with a provided constant value, or using the statistics (mean, median or most frequent) of each column in which the missing values are located. For text column "host_about" we imputed missing values by "Unknown", for column review_scores_rating we imputed missing values by median value. Also we checked whether there are any duplicate values in the dataset.
  2. Next step of Data Analysis is Data Transformation. We renamed column "availability_365" to "days_booked" and converted datatype of column "price".
  3. Next step is Data Exploratory Analysis. We found out that the count of listings for Entire home/apt is 41224, for Private room is 27479, for Shared room is 404, for Hotel room is 244.Also we found out that Entire home/apt have the most strict cancellation policy.
